# 猪只计数监控系统

## 系统概述

本系统是一个基于深度学习的猪只实时计数监控平台，采用 **YOLO11 + BoT-SORT** 目标检测与跟踪算法，通过RTSP摄像头或本地视频文件实现猪只的自动计数。

---

## 核心功能模块

### 主程序文件

| 文件名 | 功能描述 |
|--------|----------|
| `count_ui.py` | 主程序（带GUI界面），三线程异步流水线架构 |
| `count_bot.py` | 打包版本，支持PyInstaller打包为exe |
| `count.py` | 简化版本，OpenCV窗口显示，无GUI |

### 辅助工具脚本

| 文件名 | 功能描述 |
|--------|----------|
| `抽帧脚本.py` | 从视频中随机抽取帧，用于制作训练数据集 |
| `定帧分析.py` | 抽取指定帧进行模型推理并画框保存 |
| `重命名脚本.py` | 将抽取的帧随机重命名为4位数字 |
| `make_dataset.py` | 自动生成YOLO格式数据集（图片+标注） |
| `build_exe.py` | PyInstaller打包脚本，生成可执行文件 |

## 系统架构

### 三线程异步流水线设计

```
采集线程 → 推理队列 → 推理线程 → 结果队列 → 主线程显示
   ↓           ↓           ↓           ↓           ↓
RTSP/文件   非阻塞提交   GPU推理    非阻塞提交   HUD绘制+计数
```

**采集线程**：负责从RTSP摄像头或视频文件读取帧，可选录制原始视频

**推理线程**：使用YOLO模型进行目标检测和BoT-SORT跟踪

**主线程**：处理追踪结果、执行计数逻辑、绘制HUD、显示画面

## 计数算法

### 越线计数原理

- 在画面中设置一条斜线（可调顶端/底端x坐标百分比）
- 猪只从左向右越过线：C值+1
- 猪只从右向左越过线：C值-1
- C值 = 净通过数量

### ID缝合机制（防止漏计）

- 当猪只被遮挡后重新出现时，系统尝试缝合ID
- 缝合条件：
  - 方向夹角 < 30°
  - 丢失时间 < 3秒
  - 速度一致性检查
- 确保短暂遮挡不会导致重复计数

### 清零逻辑

- 支持手动清零按钮
- 支持鼠标右键三连击清零（无UI模式）
- 支持跨日自动归零
- 停止时自动记录历史并归零

## UI界面功能

### 左侧控制面板

- **视频源设置**：RTSP地址输入、本地文件选择、快速切换按钮
- **模型参数**：置信度阈值滑块、IoU阈值滑块、计数线位置调整
- **录像设置**：保存路径、推理前/后录制模式选择
- **控制按钮**：开始、停止、清零
- **计数历史**：10列×15行网格，显示历史C值记录

### 右侧显示区域

- 4:3比例视频画布，实时显示检测结果
- 轨迹绘制：绿色渐变线条显示猪只运动轨迹
- ID标注：每个检测目标显示唯一ID

### 底部状态栏

- 系统状态、C值、FPS、丢帧统计、总数、时间

## 视频源支持

### RTSP摄像头

- 默认地址：`rtsp://admin:@192.168.7.128:554/1/1`
- 使用FFmpeg子进程拉流，比OpenCV更稳定
- 支持TCP传输，自动重连机制

### 本地视频文件

- 支持格式：mp4、avi、mkv、mov
- 使用OpenCV FFMPEG后端读取
- 按原始帧率播放

---

## 视频录制

### 录制模式

- **推理前录制**：采集线程直接录制原始帧（不带HUD）
- **推理后录制**：主线程录制带HUD标注的帧

### 录制特性

- 自动分段：默认每10分钟一个文件
- 文件名格式：`YYYYMMDD_HHMMSS.mp4`
- 保存目录：默认`D:/video`

---

## Web服务

- 内置MJPEG HTTP服务，默认端口5000
- 访问地址：`http://localhost:5000`
- 提供`/video_feed`接口，支持多客户端同时查看
- HTML页面直接嵌入视频流

---

## 模型配置

### 默认模型

- 路径：`src/best.pt`
- 输入尺寸：1280×960
- 支持的模型变体：
  - `best11s896x672.pt` - YOLO11s，896×672
  - `best26n1280.pt` - YOLO26n，1280
  - `best26m576x768.pt` - YOLO26m，576×768

### BoT-SORT跟踪器

- 配置文件：`botsort.yaml`
- 关键参数：
  - `track_high_thresh`: 0.55
  - `new_track_thresh`: 0.25
  - `track_buffer`: 60帧

## 命令行参数

```bash
python count_ui.py [选项]

--rtsp          RTSP流地址或视频文件路径
--model         模型文件路径 (.pt)
--tracker       跟踪器配置文件 (yaml)
--imgsz-w       模型输入宽度 (默认1280)
--imgsz-h       模型输入高度 (默认960)
--save          视频保存目录
--duration      视频切片时长秒 (默认600)
--conf          置信度阈值 (默认0.25)
--iou           NMS IoU阈值 (默认0.6)
--device        推理设备 (0/cpu)
--web-port      Web服务端口 (默认5000)
--no-ui         无UI模式，使用OpenCV窗口
```

## 性能指标

- 推理速度：约30fps（GPU加速）
- 视频源：30fps RTSP流
- 丢帧检测：实时监控采集/推理/显示各阶段丢帧率
- 内存优化：帧缓存机制，保留最近2秒帧数据

---

## 注意事项

1. 首次运行需要CUDA环境支持
2. RTSP摄像头需确保网络稳定
3. 模型文件需放置在`src/`目录下
4. 录像保存目录需有足够磁盘空间
5. 计数线位置需根据实际摄像头视角调整
